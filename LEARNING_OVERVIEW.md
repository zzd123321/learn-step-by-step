# 学习总览

本仓库围绕“前端开发者向全栈 + AI 应用开发转型”进行长期、分路线、可复盘的学习。

## 五条学习主线

### 1. TypeScript / Vue 3

- 目录：`tracks/01-typescript-vue3/`
- 目标：从已有 Vue 2 和 Webpack 经验迁移到 Vue 3、Composition API、TypeScript 类型建模和现代前端工程化。
- 边界：聚焦前端语言能力、组件设计、状态管理、构建工具和前端项目结构，不展开后端框架细节。

### 2. Java / Spring Boot

- 目录：`tracks/02-java-springboot/`
- 目标：建立 Java 基础、面向对象思维、Spring Boot Web 开发、接口设计、分层架构和后端工程化能力。
- 边界：聚焦 Java 后端主线，不深入 Python 服务或 AI 应用实现。

### 3. Python / FastAPI

- 目录：`tracks/03-python-fastapi/`
- 目标：掌握 Python 基础语法、虚拟环境、FastAPI 接口开发、异步处理和轻量后端服务搭建。
- 边界：聚焦 Python Web 服务，不承担数据库系统化学习和 AI 应用编排主线。

### 4. 数据库 / 网络

- 目录：`tracks/04-database-network/`
- 目标：系统学习 SQL、关系型数据库、索引、事务、HTTP、网络排查、部署与工程化基础。
- 边界：作为跨项目基础能力路线，不直接替代具体语言框架课程。

### 5. AI 应用开发

- 目录：`tracks/05-ai-application/`
- 目标：学习大模型 API 调用、Prompt 设计、RAG、工具调用、流式输出、AI 产品工程化和真实业务集成。
- 边界：聚焦 AI 应用层，不训练基础模型，不深入机器学习数学推导。

## 统一课程写法

每节课采用简洁、知识点导向的写法，参考 Vue、TypeScript、Java 等官方学习文档：围绕一个知识点，讲清概念，给出最小示例，并对代码和关键细节做详细解析。

课程不强制套固定栏目，但必须保证知识点准确、代码解释充分、验证结果真实。运行方式、常见错误、练习和下一步建议按主题需要补充。

## 建议学习顺序

TypeScript/Vue 3 → Java/Spring Boot 与数据库 → Python/FastAPI → 网络与工程化 → AI 应用开发

说明：

- TypeScript/Vue 3 优先承接已有前端经验，降低迁移成本。
- Java/Spring Boot 与数据库共同建立传统后端基础。
- Python/FastAPI 用于补齐轻量服务、脚本和 AI 应用常用后端能力。
- 网络与工程化贯穿接口联调、部署、排错和系统稳定性。
- AI 应用开发建立在前后端、接口、数据和工程化基础之上。

## 协作规则

- 每条学习主线对应独立目录。
- 各线程不应同时写入同一个文件。
- 单次教学只推进一个知识点或小模块。
- 已验证结果必须来自真实执行过的命令。
