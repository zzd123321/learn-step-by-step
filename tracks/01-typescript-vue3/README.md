# TypeScript + Vue 3 + Node.js 全栈基础路线

本路线面向已有 Vue 2、Element UI、Webpack 和接口联调经验的前端开发者，目标是从“能写 JavaScript”升级到“能用 TypeScript 构建可维护的 Vue 3 与 Node.js 应用”。

## 学习原则

- 每次只推进一个核心知识点，先理解，再运行，再复盘。
- 所有示例优先使用最小可运行项目，避免为了学习概念安装大量依赖。
- 每节课都会连接真实业务场景，尤其关注接口数据、组件状态、表单、权限、错误处理和可维护性。
- TypeScript 术语会保留英文原文，并配套中文解释。

## 分阶段路线

### 阶段 1：TypeScript 的价值与运行机制

1. TypeScript 到底解决了什么问题：从 JavaScript 的运行时错误到类型系统
2. TypeScript 编译流程：源代码、类型检查、编译产物与运行时代码
3. `tsconfig.json`：如何控制项目的类型检查边界
4. 类型标注与类型推断：什么时候写类型，什么时候交给编译器

### 阶段 2：基础类型与数据建模

1. 基础类型：`string`、`number`、`boolean`、`null`、`undefined`
2. 数组、元组与只读数组：列表数据与固定结构数据
3. 对象类型：从接口返回值到页面状态
4. 字面量类型：把业务状态限制在可控范围内
5. 函数类型：参数、返回值、回调与异步函数

### 阶段 3：组合类型与类型边界

1. `interface`：描述对象契约
2. `type`：给类型表达式命名
3. 联合类型：一个值可能属于多种情况
4. 交叉类型：组合多个能力或字段
5. 可选属性与只读属性：真实业务数据的不完整与不可变约束
6. `any`、`unknown` 与 `never`：类型系统的入口、边界和不可能状态

### 阶段 4：类型收窄与安全分支

1. `typeof`、`in`、相等判断与控制流分析
2. 自定义类型守卫：把运行时判断反馈给类型系统
3. 判别联合：用业务状态驱动安全渲染
4. 穷尽检查：用 `never` 捕获遗漏分支

### 阶段 5：泛型与类型工具

1. 泛型函数：让类型像参数一样复用
2. 泛型约束：允许灵活，但保留必要字段
3. 泛型接口与泛型类型别名
4. 条件类型、映射类型与 `infer`
5. 常用 Utility Types：`Partial`、`Pick`、`Omit`、`Record`、`ReturnType`

### 阶段 6：面向对象与工程边界

1. 类、构造函数与实例类型
2. `public`、`private`、`protected` 与 `readonly`
3. 抽象类与接口的取舍
4. 装饰器的使用边界：框架能力，不是默认选择

### 阶段 7：模块系统、包管理、错误处理与测试

1. ESM 与 CommonJS 的差异
2. TypeScript 中的模块解析
3. 包管理、脚本命令与依赖边界
4. 错误处理：从 `try/catch` 到可预期错误模型
5. 基础测试：让类型检查和自动化测试各司其职

### 阶段 8：Vue 3 Composition API 类型化

1. `ref`：基础响应式值的类型推断
2. `reactive`：对象状态与深层响应式
3. `computed`：派生状态与返回值类型
4. `watch`：监听来源、回调参数与副作用边界
5. 从 Vue 2 Options API 思维迁移到 Composition API

### 阶段 9：Vue 3 组件类型设计

1. Props：组件输入契约
2. Emits：组件输出事件契约
3. Slots：插槽内容与作用域数据
4. Expose：暴露给父组件的实例能力
5. Composable：可复用逻辑的参数、返回值与副作用

### 阶段 10：前端业务工程化

1. Vue Router：路由参数、查询参数与权限元信息
2. Pinia：Store 状态、Getter 与 Action 类型
3. 接口请求层：请求参数、响应数据与错误结构
4. DTO 与 VO：后端传输对象和前端视图对象的差异
5. 前后端类型契约：共享类型、生成类型与维护边界

### 阶段 11：Node.js 与轻量后端基础

1. Node.js 运行模型与模块系统
2. HTTP 基础：请求、响应、状态码与 Header
3. REST API：资源、路由与请求方法
4. 环境变量：配置与密钥边界
5. 日志、异常处理与统一错误响应

### 阶段 12：TypeScript 小型全栈项目实践

1. 项目结构设计：前端、后端与共享类型
2. 用户、列表或表单类业务的端到端实现
3. 接口契约与前端页面联调
4. 基础测试、构建与部署前检查
5. 复盘：从局部类型到可维护系统
