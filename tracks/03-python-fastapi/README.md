# Python / FastAPI 学习路线

## 阶段目标

从 Python 基础开始，逐步掌握脚本能力、虚拟环境、包管理、Pythonic 编程、类型标注、异步处理、测试、FastAPI 接口开发、数据库访问和后端工程化，最终能够为真实全栈项目与 AI 应用提供稳定的后端服务。

## 分阶段路线

### 阶段 1：运行环境与基础语法

目标：理解 Python 解释器、虚拟环境、包管理和脚本执行方式，并掌握变量、基础类型、容器类型、控制流与函数。

- Python 解释器、执行模型、虚拟环境和包管理
- 变量、数字、字符串、布尔值、`None`
- 列表、元组、字典、集合
- 条件分支、循环、函数与返回值
- 用 JavaScript 对照理解 Python 的语法边界

### 阶段 2：函数进阶与 Pythonic 思维

目标：能写出清晰、可维护的 Python 函数和模块，理解作用域、闭包、装饰器与 lambda 的合理使用边界。

- 作用域、闭包和函数作为一等对象
- lambda：适合短表达式，不适合复杂业务逻辑
- 装饰器：把横切逻辑包在函数外层
- 模块、包、导入路径和项目结构
- 异常处理、上下文管理器、文件 I/O 与常用标准库

### 阶段 3：面向对象、协议与类型系统

目标：理解 Python 对象模型，用类型标注提升后端代码的可读性和工具支持。

- 类、实例属性、类属性和继承
- MRO：方法解析顺序，即多继承时查找方法的顺序
- 魔术方法：让对象支持内置语法和协议
- 迭代器、生成器、推导式和可迭代协议
- 类型标注、`dataclass`、Pydantic 数据模型

### 阶段 4：并发、测试与工程化

目标：具备独立组织 Python 后端项目的能力，能测试、记录日志、管理配置和控制依赖。

- `asyncio`、协程、任务和并发控制
- 线程池与进程池的适用场景
- `pytest` 单元测试和接口测试
- 日志、配置、依赖管理和目录分层
- 真实项目中的可维护性与可观测性

### 阶段 5：FastAPI 与服务端接口

目标：能用 FastAPI 构建类型清晰、校验完整、可测试的 HTTP API。

- 路由、请求参数、请求体和响应模型
- 依赖注入：把认证、数据库会话等公共能力交给框架管理
- 异常处理、错误响应和状态码设计
- Pydantic 校验与序列化
- 面向前端调用的 API 契约设计

### 阶段 6：数据库、认证与后台能力

目标：掌握服务端常见基础设施，让 API 接近真实生产项目。

- 数据库访问、ORM、迁移和事务
- 认证与授权基础
- 后台任务、缓存和消息队列入门
- 部署基础、环境变量和健康检查
- 为 AI 应用封装模型调用、任务队列和数据处理接口

### 阶段 7：Python 小型 API 项目

目标：完成一个小型但工程化的 API 项目，把 Python 基础、FastAPI、数据库、测试和 AI 应用服务串起来。

- 项目初始化与分层目录
- 接口设计、数据模型和数据库迁移
- 自动化测试、日志和配置
- 本地运行与部署说明
- 面向前端和 AI 调用方的接口文档
