# AI 应用开发学习路线

本路线面向已经具备前端基础、正在补齐 Java、Python、数据库和后端能力的学习者。目标不是“会调用一个聊天接口”，而是逐步具备独立设计、开发、测试、评估、部署和维护 LLM 应用的工程能力。

## 学习目标

- 理解 LLM、Token、上下文窗口、推理参数、系统提示词等基础机制。
- 掌握 Prompt Engineering，并能把提示词设计转化为可测试、可维护的工程接口。
- 能用 Python/FastAPI、Java 后端和 Vue 前端集成 AI 聊天、流式响应和知识库功能。
- 能构建 RAG、工具调用、Agent、MCP 等常见 AI 应用架构。
- 能处理评测、观测、安全、成本控制、版本管理和项目落地问题。
- 最终完成一个完整、可运行、可复盘的 AI 全栈项目。

## 学习原则

- 每次只推进一个核心知识点，先理解原理，再做最小实现，最后讨论工程边界。
- 对会快速变化的模型 API、SDK、协议和参数名，优先核对官方文档或本地已安装版本。
- 真实 API Key、Token、账号信息永远不写入仓库；示例只提供 `.env.example`。
- 示例优先可本地运行；如涉及真实 API，必须明确环境变量、安全边界、成本风险和验证结果。
